HTML

  • The DOCTYPE html declaration tells the browser what document type it should expect when it's opened.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The section element defines a section in a document that contains thematic content.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS will almost always follow a basic pattern like... Selector { property : Value ; } for a real example... h1 { color: purple ; }
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The best way to incorporate CSS into HTML is w an External Stylesheet, you code all the CSS in its own document and then link it to your HTML. The link should be in the head of your HTML file after the last meta tag.
  • RGB values on CSS range from 0 (off) to 255 (max). For example rgb (255,0,0) is max red while rgb (0,0,0) is black.
  • Adjacent selector example. h1+p { color: red;} This selects only the paragraphs that are immediately preceded by an h1.
  • Direct Child example div > li {color: white;} Selects only the li's that are direct children of a div element.
  • Attribute selector example... input [type="text"] {width: 300 px; color: yellow;} selects all input elements where the type attribute is set to "text".
  • Pseudo Classes are keywords which are added to a selector that specifies a special state of the selected element(s). Examples are :active, :checked, :first, :first-child, :hover, :not(), :nth-child(), :nth-of-type(). Thet always start w a colon.
  • Pseudo Elements are keywords added to a selector that lets you style a particular part of selected element(s). For example, ::after, ::before, ::first-letter, ::first-line, ::selection. generally you should use 2 colons but most browsers don't actually care, 1 can be used.
  • The Cascade in CSS refers to the fact that the order your styles are declared in and linked to matters! For example h1 {color: red;} h1 {color: purple;} PURPLE WINS!
  • Specificity is how the browser decides which rules to apply when multiple rules could apply to the same element. The more specific selector wins! The general formula to specificity goes as follows. ID> CLASS> ELEMENT
  • Border Properties: The most important ones are Border-Width, controls the thickness of the border. Border-Color, controls the color of the border. Boyder-Style, controls the line style - dashed, solid, etc.
  • Border shorthand allows you to change the width, style, and color of a border all by typing border: width that you want, style that you want, color that you want.
  • border-radius: Rounds the corners of an elements outer border edge.
  • The shorthand for padding is just padding: example. padding: 10px applies 10px of padding to all four sides!
  • EM's are relative units. With font-size, 1em equals the font-size of the parent. 2em's is twice the font-size of the parent, etc. With other properties, 1em is equal to the computed font-size of the element itself.
  • REM's pr ROOT EM's are relative to the root html elements font-size. Often easier to work with. If the root font-size is 20px, 1 rem is always 20px, 2rem is always 40 px,etc.
  • The a in rgba handles the opacity of whatever you are applying rgb on. It ranges from 0-1 with 0 being completely transparent & 1 not being transparent at all.

Git

  • GitHub is a distributed version control software. Version control is a way to save changes over time without overwriting previous versions
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone [url]: Download a repository that already exists on GitHub, inluding all of the files, branches, and commits.
  • git branch: This shows the existing branches in your local repository.
  • git commit -m "descriptive message": Records file snapshots permanently in the version history.
  • git pull: Updates your current local working branch with all new commits from the corresponding remote branch on GitHub.
  • git push: Uploads all local branch commits to the remote.
  • git log: Browse and inspect the evolution of the projects files.

JavaScript

  • The const keyword allows us to create a variable that can't be reassigned a value.
  • Control flow is the order in whic ha computer executes code in a script.
  • JS files are linked to an HTML document by being linked at the line right before the body closing tag.
  • The console.log method is used to output a message to the web console by adding an arument. IE console.log(topics).
  • An array is a single variable that is used to hold a group of data, typically a the data is related in some way.
  • The for loop consists of the keyword for, followed by a three-part conditional statement, and then a code block that defines the code to execute at each iterative step.
  • The code x++ is a shortcut way of writing x+1.
  • Functions are reusable blocks of code that contain instructions to perfrom a task.
